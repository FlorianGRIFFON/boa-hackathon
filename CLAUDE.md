# boa-hackathon — Agent Rules

## What this repo is

A monorepo for building and publishing weekly mobile apps using Expo + React Native. One app per week, each solving one specific problem with one hero feature. Currently publishing Android only.

## Repo structure

```
packages/
  shared-types/   — TypeScript types shared across all apps (@boa/shared-types)
  shared-hooks/   — React hooks for subscription and onboarding (@boa/shared-hooks)
  shared-ui/      — Reusable React Native components (@boa/shared-ui)
apps/
  <YYYY-WNN-app-name>/   — one Expo app per weekly build
agents/
  schemas/        — JSON schemas for inter-agent structured output
  templates/      — Files to copy when scaffolding a new app
  prompts/        — System prompts for each agent role
  workflows/      — Workflow scripts (.js) for Claude orchestration
.claude/
  rules/          — Per-topic rules applied to all agents
  workflows/      — Claude workflow scripts
```

## Package manager

Yarn workspaces (classic, v1). Always use `yarn` — never `npm install` inside this repo.

## Technology

- **Framework**: Expo SDK (latest), React Native, Expo Router (file-based routing)
- **Language**: TypeScript strict mode throughout
- **Builds**: EAS Build (cloud builds — Android AAB for production, APK for internal/preview)
- **Publishing**: EAS Submit → Google Play internal track
- **Platform**: Android-only for now. iOS config is preserved in existing apps and can be activated later.

## Rules (always apply)

All rules in `.claude/rules/` are active at all times:

- **scope-discipline**: Build only what is in the app's `SPEC.md`. No extra screens, no extra features.
- **shared-components**: Check `packages/shared-ui` before creating any new component.
- **no-commits-on-the-fly**: Stage all changes; one consolidated commit at the end.
- **code-comments**: Comment the *why*, not the *what*.

## App spec

Every app in `apps/` must have a `SPEC.md` generated by the brainstorm agent. Its schema is at `agents/schemas/app-spec.schema.json`. The builder agent reads `SPEC.md` as its sole source of requirements.

## When scaffolding a new app

1. Copy `agents/templates/metro.config.template.js` → `apps/<name>/metro.config.js`
2. Copy `agents/templates/app.config.template.ts` → `apps/<name>/app.config.ts` and fill in APP_* placeholders
3. Copy `agents/templates/eas.json.template` → `apps/<name>/eas.json`
4. Run `expo install` inside the app directory — not `yarn add`
5. The app must declare `@boa/shared-ui`, `@boa/shared-hooks`, and `@boa/shared-types` as dependencies

## Human checkpoints

The weekly workflow pauses for human review at two points:
1. After the brainstorm agent outputs the spec — human approves before build starts
2. After the QA agent outputs its report — human approves before publish starts

Do not skip or automate these checkpoints.
